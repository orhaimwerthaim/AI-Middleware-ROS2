* צריך לאפשר הפעלת קוד באמצעות (topics, services, actions)
* להגדיר אירועים: ניתן ליזום אותם ע"י א. תנאי משתנים (נבדק כל כמה זמן) ב. ע"י זמן קבוע (טרום הפעלת יכולת, לאחר סיומה, או זמן מוגדר לאחר התחלתה) או חוזר כל תדירות. ג. קבלת הודעה מtopic או service
כאשר אירוע ניזום הוא יכול א. להפסיק את היכולת. ב. לייצר עדכון התקדמות. ג. ליזום הפעלת יכולת אחרת או הפסקתה.  ד. לעדכן ערכי משתנים: משתנים נדיפים, שכבה פונקציונאלית, חשופים לסולבר (משתני תצפית או משתני מצב).

מושגים:
יוזם- השכבה שיוזמת את הפעלת הקוד, יכולה להיות סולבר, BT, FSM, או קוד או אפילו יכולת אחרת.

  ** סוגי משתנים:
  שם- external_termination_mode
  סוגי DATA: מספר שלם, דצימלי, מחרוזות, רשימות (המכילות אחד מהסוגים)
  הסבר: בסיום הפעולה יוחזר משתנה זה ליוזם החיצוני עם הערכו הנוכחי, ניתן לקבוע לו ערך התחלתי. ליכולת ניתן להגדיר מספר משתנים כאלו וכולם ישלחו ליוזם בסיום.

   שם- external_variable
    סוגי DATA: מספר שלם, דצימלי, מחרוזות, רשימות (המכילות אחד מהסוגים)
  הסבר: משתנה זה קיים אצל היוזם (למשל אצל הסולבר), מקבלים את ערכו ההתחלתי מהיוזם, יכול לשמש כהקשר נוסף בהפעלה, ניתן לעדכן אותו ולהעביר מידע ליוזם (ניתן לקבוע האם העדכון ישלח בסיום הפעולה או בזמן אמת). רשימת משתנים אלו תוגדר בAM  תשמור אותו אצלה אך תשלח עדכונים ליוזם כשערכו ישנו. יהיה ממשק לקבל את ערכי המשתנים הללו.

  שם- ros2_persistent
  סוגי DATA: מספר שלם, דצימלי, מחרוזות, רשימות (המכילות אחד מהסוגים), יכול להכיל משתנה מסוג הודעה קיימת.
  הסבר: משתנים אלו משמשים להעברת מידע בין יכולות, נשמרים ברמת ROS2.

  שם- ros2_volatile
  סוגי DATA: מספר שלם, דצימלי, מחרוזות, רשימות (המכילות אחד מהסוגים).
  הסבר: משתנים אלו נועדו לחישובים פנימיים בזמן ביצוע היכולת, נעלמים בסיומה.

  שם- skill_parameter
  סוגי DATA: מספר שלם, דצימלי, מחרוזות, רשימות (המכילות אחד מהסוגים).
  הסבר: ערך המתקבל מהיוזם ניתן להשתמש בו החל מטרום הפעלת קוד היכולת (למשל לעיבוד מקדים).

  ** אירועים:
  אירוע יכול להיות ניזום באופן הבאים-
    א. בהתקיים תנאים על משתנים (מוגדר כל כמה זמן לבדוק) ב. מיד בהפעלת המערכת (יכול לשמש להפעלת יכולות ברקע) ג. בעת הפעלת היכולת ע"י היוזם. ד. בסיום פעילות היכולת. ה. זמן מוגדר מאז שיזמו את הפעלת היכולת. ו. חוזר כל תדירות. ז. בעת קבלת הודעה מtopic מסויים תופעל. ח. כשיוחזר ערך מservice מסויים.

    אירוע יכול לגרום ל:
       א. הפסקת היכולת. ב. לייצר עדכון התקדמות (מידע או התרעה ליוזם) ג. ליזום הפעלת יכולת אחרת או להפסיקה. ד. לעדכן ערכי משתנים מכל הסוגים.


    עידכונים ליוזם:
    א. פעולה הסתיימה (ניתן לקבל את הערכים שלה).
    ב. עודכן משתנה מצב של היוזם (external_variable).
    ג. עדכון יוזם שניזום באירוע.
  file name: navigate.am
  _____________________________
  variable: success_result
  variable_type: external_termination_mode

  Activation
  ----------------------
  
  
  
  
  
  #get the location
  ros2 topic echo /turtle1/pose


# action to set theta
ros2 action send_goal /turtle1/rotate_absolute turtlesim/action/RotateAbsolute "{theta: 0.5 }"

# topic to activate action that moves the turtle forwards
ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "{linear: {x: 1.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}" --times 3
